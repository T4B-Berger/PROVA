# Respondent explorer design (phase 3)

## Objectif
Permettre une lecture réponse par réponse, structurée et filtrable, sans dévoiler de données personnelles.

## Fonctions
- filtres : rôle, usage pro, maturité, early adopters
- recherche plein texte
- navigation indexée réponse par réponse
- blocs de lecture: profil, usages, bénéfices, risques, formation, verbatims
- badge de maturité

## Règles
- anonymisation des verbatims (`[email]`, troncature)
- pas d'affichage nominatif
- en mode Description seule=Oui: strictement factuel
